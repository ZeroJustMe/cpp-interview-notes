# 02. 网络进阶：重传、拥塞控制、长连接、WebSocket

## 1. TCP 重传机制有哪些？

### 标准回答
常见包括：
- 超时重传（RTO）
- 快速重传（收到多个重复 ACK 时触发）

## 2. 什么是拥塞控制？

### 标准回答
拥塞控制是 TCP 为防止网络过载而采取的一组发送速率调节机制。

### 常见阶段
- 慢启动
- 拥塞避免
- 快速重传
- 快速恢复

## 3. 长连接和短连接区别？

### 短连接
每次请求都新建连接，用完即断开。

### 长连接
多个请求复用同一连接，减少握手和挥手开销。

### 面试场景
HTTP/1.1 默认支持长连接（Keep-Alive 语义）。

## 4. HTTP/1.1、HTTP/2、HTTP/3 有什么区别？

### HTTP/1.1
- 文本协议
- 默认长连接
- 存在队头阻塞问题（应用层）

### HTTP/2
- 二进制分帧
- 多路复用
- 头部压缩
- 同一 TCP 连接上并发多个流

### HTTP/3
- 基于 QUIC（UDP）
- 改善传输层队头阻塞问题
- 连接迁移更友好

## 5. 什么是 WebSocket？

### 标准回答
WebSocket 是在单个 TCP 连接上进行全双工通信的协议，适合实时通信场景，如聊天、推送、实时协作。

### 与 HTTP 的关系
- 建立阶段基于 HTTP Upgrade
- 升级后变成持久双向通信通道

## 6. 什么是队头阻塞？

### 标准回答
队头阻塞指前面的数据包或请求未处理完成，导致后续请求即使准备好了也无法继续处理。

### 场景区分
- HTTP/1.1 管线化问题
- TCP 丢包导致的传输层阻塞

## 7. DNS 查询过程大概是怎样的？

### 标准回答
通常包括：
1. 浏览器缓存
2. 系统缓存
3. 本地域名服务器
4. 根域名服务器
5. 顶级域服务器
6. 权威域名服务器

## 8. CDN 原理是什么？

### 标准回答
CDN 通过把内容缓存到离用户更近的边缘节点，减少跨地域传输延迟，提升访问速度和可用性。

## 9. 复习提示

这部分常出现在后端或高并发场景追问里：
- 长连接优势
- HTTP2/3 核心改进
- WebSocket 和 HTTP 关系
- DNS / CDN 基础链路
