# 03. Linux：文件系统、inode、信号、僵尸进程

## 1. inode 和文件名是什么关系？

### 标准回答
文件名是目录项里的名字，inode 保存文件元数据和数据块信息。目录本质上是“名字 -> inode 编号”的映射。

## 2. 软链接和硬链接区别？

### 硬链接
多个文件名指向同一个 inode。

### 软链接
是一个独立文件，内容是目标路径。

### 面试易错点
- 硬链接通常不能跨文件系统
- 对目录创建硬链接通常受限制

## 3. 什么是信号？

### 标准回答
信号是 Linux/Unix 中一种异步事件通知机制，用来通知进程某类事件发生，例如中断、终止、子进程状态变化等。

## 4. kill -9 和 kill -15 区别？

- `SIGTERM(15)`：请求进程正常退出，可捕获和清理资源
- `SIGKILL(9)`：强制杀死，不能被捕获或忽略

## 5. 为什么会产生僵尸进程？

因为子进程退出后，父进程没有调用 `wait/waitpid` 回收退出状态。

## 6. 面试提示

Linux 基础题不难，但很容易被问穿：
- 文件名不是文件本体
- inode / 目录项 / 链接关系要说清
- kill 信号不要混
