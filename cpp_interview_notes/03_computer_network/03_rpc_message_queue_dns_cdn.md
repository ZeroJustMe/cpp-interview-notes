# 03. RPC、消息队列、DNS、CDN 基础

## 1. RPC 和 HTTP API 区别？

### 标准回答
RPC 更强调“像调用本地函数一样调用远程服务”，通常有更强的接口定义、序列化协议和治理能力；HTTP API 更通用、开放、易调试。

## 2. 消息队列的作用是什么？

- 解耦
- 削峰填谷
- 异步处理
- 广播/订阅

## 3. 消息队列会带来什么问题？

- 消息重复
- 消息丢失
- 顺序性
- 消费积压
- 一致性复杂化

## 4. 什么是幂等？

### 标准回答
同一个操作执行一次和执行多次，结果一致，这叫幂等。

### 场景
消费消息、重试请求、支付回调等。

## 5. DNS 为什么重要？

因为它负责把域名解析为 IP，是几乎所有网络访问的入口基础设施。

## 6. CDN 为什么能加速？

通过把静态内容分发到边缘节点，缩短用户到资源的网络距离，并降低源站压力。
