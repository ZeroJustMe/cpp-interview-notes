# 01. 对象模型、内存布局、类与多态

## 1. C++ 对象在内存中长什么样？

### 标准回答
C++ 对象通常由以下几部分组成：
- 非静态成员变量
- 由于对齐产生的填充字节（padding）
- 如果类中有虚函数，通常还会有一个虚函数表指针 `vptr`

成员函数代码通常不存储在对象实例中，而是在代码段中，由所有对象共享。

### 深挖细节
1. **空类大小为什么不是 0？**
   - 空类通常 `sizeof` 为 1。
   - 原因是标准要求不同对象有不同地址表示。

2. **静态成员变量算不算对象大小？**
   - 不算。
   - 静态成员变量属于类，不属于某个对象实例。

3. **成员函数算不算对象大小？**
   - 不算。函数代码在代码段。

4. **虚函数为什么会影响对象大小？**
   - 因为通常要增加一个 `vptr`，在 64 位平台通常占 8 字节。

## 2. 类对象大小如何计算？

### 标准回答
对象大小 = 各成员大小 + 对齐填充 + 继承带来的基类子对象大小 + 可能存在的虚表指针。

### 例子
```cpp
class A {
    char c;
    int i;
};
```
`sizeof(A)` 往往不是 5，而可能是 8，因为 `int` 对齐到 4 字节。

### 面试追问
#### 为什么调整成员顺序会影响大小？
因为编译器要满足对齐要求，成员顺序改变会影响 padding 分布。

#### 对齐的意义是什么？
- 提高 CPU 访存效率
- 减少未对齐访问代价
- 某些体系结构上未对齐访问甚至会异常

## 3. 什么是内存对齐？

### 标准回答
内存对齐指编译器按类型对齐要求为对象分配内存地址，使其地址满足特定倍数边界，从而提升访问效率。

### 常见规则
- 成员偏移量通常是该成员对齐要求的整数倍
- 结构体整体大小通常是其最大对齐数的整数倍

### 易错点
- 不要把“对齐”理解成语言语义，它更偏 ABI / 编译器实现与平台协同
- 不同编译器、平台结果可能不同

## 4. 什么是虚函数？怎么实现动态多态？

### 标准回答
虚函数用于支持运行时多态。常见实现方式是：
- 类含虚函数时，编译器为类生成虚函数表 `vtable`
- 对象中存放指向虚表的指针 `vptr`
- 通过基类指针或引用调用虚函数时，运行时根据对象实际类型在虚表中查找对应函数入口

### 深挖细节
#### 动态绑定和静态绑定的区别
- 静态绑定：编译期决定调用目标，比如普通函数、重载
- 动态绑定：运行期根据实际对象类型决定调用目标，比如虚函数

#### 虚函数的代价
- 对象体积变大（多一个 `vptr`）
- 调用多一次间接跳转
- 可能影响内联优化

## 5. 构造函数为什么不能是虚函数？

### 标准回答
构造函数不能是虚函数，因为构造阶段对象尚未完成，动态类型还在建立过程中，无法对“尚未构造完成的对象”做合理的虚派发。

### 深挖细节
- 构造派生类对象时，会先构造基类，再构造派生类
- 在基类构造阶段，对象还不能被视作完整派生类对象
- 如果允许构造函数虚调用，语义会非常混乱

## 6. 析构函数为什么常常要设为虚函数？

### 标准回答
如果一个类会被当作多态基类使用，那么析构函数应该定义为虚函数。这样通过基类指针删除派生类对象时，才会先调用派生类析构，再调用基类析构，避免资源泄漏。

### 例子
```cpp
class Base {
public:
    virtual ~Base() {}
};
class Derived : public Base {
public:
    ~Derived() {}
};
Base* p = new Derived();
delete p;
```

### 易错点
- 不是“所有类析构都必须虚”
- 只有“可能通过基类指针删除派生对象”的类才应优先考虑虚析构

## 7. 什么是纯虚函数和抽象类？

### 标准回答
纯虚函数是使用 `= 0` 声明的虚函数，包含纯虚函数的类称为抽象类，抽象类不能实例化。

### 示例
```cpp
class Shape {
public:
    virtual void draw() = 0;
};
```

### 追问点
- 抽象类可以有成员变量和普通成员函数
- 纯虚析构函数也可以有函数体定义

## 8. 单继承、多继承、虚继承有什么区别？

### 标准回答
- **单继承**：一个类只继承一个基类
- **多继承**：一个类继承多个基类
- **虚继承**：为了解决菱形继承中公共基类重复拷贝问题

### 深挖细节
#### 菱形继承问题
```cpp
class A {};
class B : public A {};
class C : public A {};
class D : public B, public C {};
```
`D` 中会有两份 `A` 子对象。

#### 虚继承解决方式
通过虚继承让最底层对象只保留一份公共基类子对象，但实现更复杂，访问成本更高。

## 9. override 和 final 的作用是什么？

### 标准回答
- `override`：显式声明该函数重写基类虚函数，若签名不匹配编译报错
- `final`：禁止进一步重写或继承

### 面试价值
它们不是“语法糖”，而是帮助在编译期发现多态接口错误的重要手段。
