# 03. Redis：过期淘汰、持久化、高可用

## 1. Redis 过期删除策略有哪些？

### 标准回答
- 定时删除
- 惰性删除
- 定期删除

实践中通常结合使用，以平衡 CPU 开销和内存回收及时性。

## 2. Redis 内存淘汰策略有哪些？

### 常见策略
- noeviction
- allkeys-lru
- volatile-lru
- allkeys-lfu
- volatile-ttl
等

## 3. Redis 为什么会有大 key 问题？

### 风险
- 单次操作阻塞更久
- 网络传输耗时大
- 删除成本高
- 主从复制压力大

## 4. Redis 主从复制有什么作用？

- 读扩展
- 故障恢复基础
- 数据备份冗余

## 5. Sentinel 和 Cluster 区别？

### Sentinel
偏高可用，负责监控、故障转移，不做数据分片。

### Cluster
同时解决高可用和数据分片问题。

## 6. 缓存一致性怎么做？

### 常见策略
- 先更新数据库，再删缓存
- 延迟双删
- 基于消息队列异步失效
- 业务层幂等和兜底校验

## 7. 面试提示

Redis 不止会问“为什么快”，还会问：
- 高可用怎么做
- 大 key 怎么治理
- 缓存一致性如何兜底
