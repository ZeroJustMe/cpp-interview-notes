# 01. 设计模式、系统设计、高并发项目问答

## 1. 单例模式怎么写？有什么问题？

### 常见写法
```cpp
class Singleton {
public:
    static Singleton& instance() {
        static Singleton s;
        return s;
    }
private:
    Singleton() = default;
};
```

### 标准回答
局部静态对象版本是现代 C++ 中常见且推荐的懒汉单例写法，线程安全由标准保证初始化一次。

### 面试追问
单例的问题：
- 全局状态难测试
- 生命周期不透明
- 容易形成隐藏依赖
- 在大型系统中可能影响可维护性

## 2. 工厂模式有什么价值？

### 标准回答
工厂模式把对象创建逻辑和使用逻辑分离，降低耦合，提升扩展性。

### 常见场景
- 根据配置创建不同策略类
- 插件式组件加载
- 协议解析器、连接器、消息处理器

## 3. 观察者模式适合什么场景？

### 标准回答
当一个对象状态变化需要通知多个依赖对象时，可以使用观察者模式。

### 工程场景
- 事件总线
- 配置更新广播
- UI/消息订阅

### 注意
在 C++ 中要格外注意对象生命周期和订阅关系，防止悬垂回调。

## 4. 生产者消费者模型怎么实现？

### 标准回答
典型实现是：
- 一个线程安全队列
- 条件变量协调
- 生产者写入任务
- 消费者阻塞等待并取任务执行

### 追问点
- 队列满了怎么办
- 如何优雅退出
- 多生产者多消费者如何减少锁竞争

## 5. 线程池设计要点有哪些？

### 标准回答
- 任务队列
- worker 线程集合
- 条件变量/信号量唤醒
- 停止标志
- 拒绝策略
- 任务异常处理

### 面试高分点
如果你能额外说到：
- 任务窃取
- 动态扩缩容
- 优先级队列
- 长短任务隔离
通常就比普通答案更强。

## 6. 高并发系统设计要关注什么？

### 标准回答
常见关注点：
- 限流
- 熔断
- 降级
- 超时控制
- 重试策略
- 线程池隔离
- 缓存设计
- 数据一致性
- 监控与告警

## 7. 如何设计一个高性能 C++ 服务器？

### 一个比较像面试的答法
1. 网络层：非阻塞 socket + epoll
2. 并发层：IO 线程 + 业务线程池
3. 内存层：对象池 / 合理的内存管理，减少频繁分配
4. 协议层：明确消息边界，避免粘包拆包问题
5. 稳定性层：超时、限流、熔断、日志、监控
6. 数据层：缓存、本地队列、异步落盘/数据库

## 8. 如果线上 CPU 飙高，你怎么排查？

### 面试回答思路
- 先看整体监控：CPU、负载、QPS、RT、错误率
- 再看进程/线程维度热点
- 用 perf/top/gdb/火焰图定位热点函数
- 分析是业务死循环、锁竞争、自旋、频繁拷贝，还是系统调用过多
- 结合最近发布、流量变化、异常日志回溯

## 9. 如果线上内存持续上涨，你怎么排查？

### 思路
- 区分内存泄漏、缓存膨胀、碎片、对象池回收不及时
- 看 RSS、堆增长、对象数量
- 分析最近变更
- 工具层面可用内存分析器、日志埋点、采样定位

## 10. 项目问题怎么回答更像“会做事的人”？

### 建议结构
- 背景：业务场景和瓶颈是什么
- 问题：为什么原方案不够
- 方案：你做了哪些改动
- 结果：性能、稳定性、资源成本改善多少
- 取舍：方案有什么副作用、未来还能怎么优化

## 11. 面试陷阱

### 不要空讲模式
设计模式不是背名字，而是：
- 解决什么问题
- 为什么要这样拆
- 有什么成本
- 在什么场景下不适合
